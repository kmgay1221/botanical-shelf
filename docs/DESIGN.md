# DESIGN.md — 植物棚（BOTANICAL SHELF）

AIコーディングエージェント向けデザイン仕様書。実装時は本書と `docs/plant-ui-mockup-full.html`（確定モックアップ）を正とし、モックの見た目を忠実に再現すること。迷ったらモックのCSSを参照する。

## 1. デザインコンセプト

**「夜の温室ギャラリー」**。アガベ趣味家の株を美術館の収蔵品のように見せる、ダーク基調の高級感あるUI。装飾は最小限に抑え、株の写真と「年間水やりリズムバー」（署名要素）に視線を集める。道具としては即断即決（開いた瞬間に今日の水やりが分かる）を最優先する。

## 2. カラートークン

```css
:root{
  --bg:       #0f1411;  /* 背景。純黒ではなく深い緑墨色 */
  --surface:  #171e19;  /* カード */
  --surface2: #1e2721;  /* モーダル・浮き要素 */
  --line:     #2a352d;  /* 罫線・ボーダー */
  --ink:      #e8ede9;  /* 主文字 */
  --ink2:     #93a096;  /* 副文字 */
  --ink3:     #5d6a61;  /* 補助・ラベル */
  --glaucous: #a9c6b4;  /* 主アクセント: チタノタの白青い葉色。当月・選択・主ボタン */
  --water:    #7fb2c9;  /* 水やり系アクション・期日 */
  --moon:     #b3a8c9;  /* 休眠・断水 */
  --sun:      #d3b47e;  /* 図鑑・管理者・注意 */
}
```

- 禁止: 純黒背景（#000）、蛍光グリーン/ビビッドな彩度の高いアクセント、白背景への反転
- 状態の意味色は固定: 水＝water、休眠＝moon、収集/特別＝sun。用途を混在させない

## 3. タイポグラフィ（日本語）

```css
--serif: "Shippori Mincho","Hiragino Mincho ProN","Yu Mincho",serif;
--sans:  "Zen Kaku Gothic New","Hiragino Kaku Gothic ProN","Noto Sans JP",sans-serif;
```

- Google Fontsから Shippori Mincho (500/600) と Zen Kaku Gothic New (400/500/700) を読み込む（next/font/google 使用）
- **明朝（serif）の用途**: 画面タイトル、株名（プレート）、数値の見出し（ストリーク日数等）のみ。本文には使わない
- **ゴシック（sans）**: それ以外すべて
- 見出し: 19–24px / letter-spacing .06–.12em。英字アイブロウ: 10px / letter-spacing .28em / uppercase / --ink3
- 本文: 12–13px / line-height 1.6–1.8。補助: 9.5–11px
- 学名は必ずイタリック・--ink2（例: *Agave titanota* 'Hakugei'。品種名クォート部は立体でも可）

## 4. レイアウト・形状

- モバイルファースト。コンテンツ幅は画面いっぱい、左右パディング 18px
- 角丸: カード 18px / 入力・ボタン 14px / チップ 999px / サムネイル 14px / モーダル上部 24px
- 影は多用しない。ダークでは境界を --line の1pxボーダーで作る
- ボトムナビ5タブ固定: 今日 / 植物棚 / 追加 / 図鑑 / 設定。アクティブは --glaucous

## 5. コアコンポーネント

### 5.1 ミュージアム・プレート（placard）
株名の標準表示。左に2pxの縦線（--glaucous、休眠株は --moon）、和名は明朝13.5px、直下に学名イタリック10px。全画面で共通使用。

### 5.2 年間水やりリズムバー（署名要素）
- 12本のバー。高さ＝水やり頻度（間隔が短い月ほど高い）。計算: `height% = minInterval / interval * 100`
- 当月: --glaucous＋わずかなグロー。他月: #33463a。断水月(null): 高さ3–6pxの #2a2537（紫がかった低いバー）
- カード内は高さ16px、詳細画面は64px＋月ラベル（1–12）＋説明文
- このコンポーネントは shelf カード / 株詳細 / AI生成プレビュー / 図鑑 で共通化する

### 5.3 サムネイル
`radial-gradient(circle at 50% 20%, #243129, #151c17 70%)` の台座に写真 or 種イラストSVG。休眠中は `grayscale(.5) brightness(.55)`＋右下に「休眠中」9px表示。左上にDAYバッジ（半透明黒地、backdrop-blur）。

### 5.4 チップ / ボタン / 入力
- チップ: 999px角丸、1pxボーダー、10px。状態色は §2 に従う
- 主ボタン: --glaucous地に #10160f 文字、太字。水やり系主ボタンのみ --water 地
- 完了ボタン: 44px円形、--waterのアウトライン → 完了で塗り＋✓＋リップル演出
- 入力: --surface地、--lineボーダー、13px。プレースホルダーは --ink3

### 5.5 セグメント / ステッパー / トグル
モックの .seg / .stepper / .toggle をそのまま踏襲。選択状態は --glaucous 塗り。

## 6. モーション

- 水やり完了: リップル（0.7s ease-out、外周へ拡散）
- 画面遷移・リスト表示に過剰なアニメーションを入れない。ホバー/タップは opacity か軽い scale のみ
- `prefers-reduced-motion: reduce` では全アニメーションを無効化

## 7. ライティング（UI文言）

- トーン: 静かで丁寧、少しだけ詩的。絵文字はアイコン代替として最小限（💧🌱💡🌙📖）
- ボタンは動詞で結果を言い切る:「水やりを記録」「植物棚に迎える」「ログインリンクを送る」
- 遊び心の定型文（変更しない）: 湿度高めの日「今日は湿度が高め。用土の乾きを確認してから」/ 断水明け「（株名）が目覚めました」
- 空状態は次の行動を示す: 例「まだ株がいません。＋から最初の一株を迎えましょう」
- エラーは謝らず、原因と対処を短く

## 8. 画面別の要点（詳細はモック参照）

| 画面 | 要点 |
|---|---|
| ログイン | 中央寄せ。ロゴ（アガベロゼットSVG）＋「植物棚」明朝 letter-spacing .3em。招待制の注記 |
| ホーム | 日付＋DAY → 見出し「今日の水やり n株」→ 天気注記バナー →「すべて完了」ボタン → タスクカード列 → 実生トレイ → 休眠中 → ストリーク2枚 |
| 植物棚 | 絞り込みチップ横スクロール → 2列グリッド。カード＝サムネ/プレート/リズムバー/期日チップ |
| 株詳細 | ヒーロー写真 → プレート＋タグ → リズムパネル（大）→ 3アクション → 成長ギャラリー → タイムライン |
| 記録 | ボトムシート。アクション切替セグメント→日時→写真→メモ→保存 |
| 追加① | 検索＋結果リスト＋「AIに調べてもらう」破線ボックス |
| 追加② | AI生成プレビュー: プレート＋確度チップ（sun色）＋リズム＋サイズ切替セグメント＋登録/再生成 |
| 追加③ | 写真アップロード（破線・プレースホルダー説明）→ニックネーム→サイズ4区分→置き場所→メモ→「植物棚に迎える」 |
| 図鑑 | カテゴリチップ→収集率バー→3列グリッド。未収集は brightness(.28) grayscale のシルエット |
| 設定 | 行リスト＋トグル。ADMINチップ（sun）、iOSホーム画面追加ガイドカード |

## 9. Do / Don't

- Do: 余白と1px罫線で品を出す。数値は明朝で見せ場を作る。リズムバーを全画面で一貫させる
- Do: 写真未登録はカテゴリ別イラストSVG（モックの agave() 生成ロジックを流用）
- Don't: 白背景の画面を作らない。カードに重い影やグラデ枠を付けない。角丸やアクセント色を画面ごとに変えない
- Don't: ホームに植物棚グリッドを混在させない（役割分離が確定事項）
- アクセシビリティ: フォーカスリング可視（--glaucous 1.5px）、コントラストは本文4.5:1以上を保つ、タップ領域44px以上
